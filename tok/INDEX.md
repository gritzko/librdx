# tok/ — Lightweight ragel-based tokenizers

Produces the same tag classification as ast/ tree-sitter parsers
(D=comment, G=string, L=number, R=keyword, P=punct, H=preproc, S=default)
but without the tree-sitter dependency.

## Headers

| Header | Purpose |
|--------|---------|
| TOK.h  | Common callback typedef `TOKcb`, dispatch API `TOKLexer()` |
| CT.h   | C tokenizer: `CTstate`, `CTLexer()` |

## Files

| File      | Purpose |
|-----------|---------|
| TOK.c     | Extension-to-lexer dispatch (c/h/cpp/hpp → CT) |
| CT.c      | Callbacks: keyword lookup, tag mapping |
| CT.c.rl   | Ragel source (hand-written scanner using `|*` `*|`) |
| CT.rl.c   | Generated by: `ragel -C CT.c.rl -o CT.rl.c -L` |

## Tag mapping

| Callback    | Tag | Meaning |
|-------------|-----|---------|
| CTonComment | D   | Comment (gray) |
| CTonString  | G   | String literal (green) |
| CTonNumber  | L   | Number literal (cyan) |
| CTonPreproc | H   | Preprocessor directive (pink) |
| CTonWord    | R/S | Keyword (red) or identifier (default) |
| CTonPunct   | P   | Punctuation (gray) |
| CTonSpace   | S   | Whitespace (default) |

## Regenerating

```sh
cd tok/
ragel -C CT.c.rl -o CT.rl.c -L
```

## Future tokenizers

Follow the `CT` pattern: `GOT` (Go), `PYT` (Python), `JST` (JavaScript), `RST` (Rust), etc.
